# Contributing to the OpenArbor Alliance

> *"We don't need users. We need guardians of the canopy."*

The OpenArbor Alliance is governed by the [Contribution Protocol](https://arboracle.app/ContributionProtocol). This document is the practical guide.

---

## Before You Contribute

1. Read the [Manifesto](https://arboracle.app/Manifesto) — understand why this exists
2. Read the [Contribution Protocol](https://arboracle.app/ContributionProtocol) — understand how we govern
3. Open a Discussion to introduce yourself — who you are, what you do with trees, why you're here

We are **human-first, narrow AI**. If you're here to use AI to generate low-quality contributions at scale, this is not your repo.

---

## Contribution Tiers

### 🌱 Citizen (Anyone)
- Document trees on [iNaturalist](https://www.inaturalist.org/projects/arboracle)
- Report bugs with clear reproduction steps
- Suggest features in Discussions
- Improve documentation

### 🌿 Steward (Field Professionals)
- Validate features against real-world workflows
- Contribute species data, IPM protocols, or ordinance information
- Test beta features and provide structured feedback
- Write field guides and how-to documentation

### 🌳 Developer (Engineers)
- Submit PRs with clear descriptions and test coverage
- Participate in architecture decisions
- Review other PRs
- Maintain open-source dependencies

### 🏛️ Researcher (Scientists & Academics)
- Contribute ecological methodology
- Validate carbon, water, and biodiversity calculation models
- Co-author documentation with scientific grounding
- Provide peer review for ecological claims

---

## Submitting a Pull Request

1. Fork the repo
2. Create a branch: `git checkout -b feature/your-feature-name`
3. Make your changes with clear, descriptive commits
4. Test thoroughly — especially field workflows on mobile
5. Submit a PR with:
   - What problem this solves
   - How you tested it
   - Any ecological or scientific context relevant to the change

---

## Code of Conduct

The only rule: **Do not commodify the commons.**

- Contributions that create proprietary lock-in will not be merged
- Contributions that degrade data sovereignty will not be merged
- Contributions that undermine ecological integrity will not be merged

Everything else is discussable.

Open a Discussion. We're a small team and a growing community. We respond to everything.
